Version 1
Build the Org X admin MVP for RiiseMap.

RiiseMap is a workforce pathway and career mobility platform. It helps organizations guide learners into tech careers through roadmaps, projects, events, coaching, community support, readiness tracking, and impact reporting.

Org X represents the organization using RiiseMap. Org X may be:
- a nonprofit
- workforce board
- college
- community organization
- employer-sponsored workforce program

The primary admin persona is Denise Carter, a workforce program manager.

Denise is:
- mission-driven
- overloaded
- moderately technical
- responsible for learner outcomes
- responsible for funder and grant reporting
- using spreadsheets, email, calendars, and CRMs today

She needs the app to feel:
- professional
- calm
- intuitive
- trustworthy
- useful every day
- low-friction
- board-ready
- grant-ready

Avoid:
- enterprise software clutter
- Salesforce-style density
- complex configuration screens
- technical jargon
- overwhelming dashboards
- raw BI-tool aesthetics

The app should feel closer to:
- Notion
- Linear
- Airtable
- Asana
- modern HubSpot
- modern nonprofit impact dashboards

Use mock data.

--------------------------------------------------
TECH STACK
--------------------------------------------------

Use a modern full-stack web app.

Recommended:
- React
- TypeScript
- Tailwind CSS
- shadcn/ui components
- Recharts for charts
- Local mock data or simple JSON data store
- Responsive desktop-first layout

Build clickable prototype behavior.

No authentication required for MVP unless easy to stub.

--------------------------------------------------
CORE PRODUCT IDEA
--------------------------------------------------

This is not just an admin dashboard.

It is a calm operational command center that helps Denise:

1. See what needs attention today
2. Support learners before they drop off
3. Track program momentum
4. Manage coaches, events, projects, and pathways
5. Measure workforce readiness
6. Create grant-ready reports
7. Turn concrete learner data into impact stories

The system should answer these questions quickly:

- Who needs support?
- Are learners progressing?
- Which programs are working?
- What outcomes can we report?
- How do we justify funding?
- What stories show real impact?

--------------------------------------------------
DATA DEFINITIONS
--------------------------------------------------

Use clear metric definitions.

Examples:

Active learner:
A learner who has logged in, completed a milestone, attended an event, or interacted with a coach in the last 14 days.

Roadmap completion:
Percentage of assigned roadmap milestones completed.

Readiness score:
Composite score based on profile completion, project completion, milestone progress, event participation, and coach readiness review.

Placement-ready:
Learner has completed required roadmap milestones, resume review, interview prep, at least one project, and coach approval.

At-risk learner:
Learner with inactivity, missed milestones, low progress, or low confidence check-ins.
